\section*{概要}
環境問題は世界規模で起きている．その問題の一つとして，個人のエネルギー消費がある．
水の出しっぱなしや，電気のつけっぱなしなどを防ぐことによりエネルギー消費を
抑えられる．
しかし，各個人はエネルギー消費について敏感ではない．24時間エネルギーが利用可能な状態に
あるため，エネルギーを尊む気持ちが薄れている．

そこで，本リポートでは，非接触型タグ\cite{autoid}を用いてリソースを消費している
個人を特定し，現在消費している環境エネルギー（リソース）をリアルタイムで
計測するシステムを提案をする．

% このシステムを友人x人に話してみた所，
% 効果が上がると答えた人はx人だった．


\section{はじめに}
近年，環境破壊などの問題が深刻化している．
我々個人ができる簡単な環境保全の方法として，
日々の生活の中でのエネルギー消費を最低限に抑えることが
一例としてある．
しかし，各個人での環境への意識は低い．環境保全への
意識を高めるために自分の使っているリソースをリアルタイムで
監視するシステムを提案する．
また，このシステム提案を他の人に聞いてもらい，
効果性の評価をしてもらう．


\section{システム構成図}
被験者となるユーザは，RFタグを携帯していることを前提とする．また，リソースには
使用メーターと，RFリーダが設置されているとする．
こうすることにより，個人が使っているリソースを逐次モニタリングできる．
モニターしたデータはデータベースに送られ，データの履歴などを保存する．
蓄積したデータを元に，ユーザに対してリソースの使用状態などをグラフィカルに
表示する．
（図\ref{flow}参照）

リソースを監視する対象としては，ガス・水・電気などがあげられる．また，屋内の
設置にとどまらず，車にもリソースモニタをつけて，ガソリン使用量の監視なども
行うとおもしろいかもしれない．

ビルなどの多数の人がいる場所ではビル内で使用しているリソース量を
居る人数で割れば個人の使用量を算出出来るだろう．

このシステムにおいて工夫する点として，ユーザに対してのリソース表示を
いかに引きつけるものであるかが重要であり，その表示からユーザが
リソースの使用に対しての意識を持ってもらうのがリポートの目的でもある．

GUIの工夫として以下のようなことが考えられる
\begin{itemize}
 \item 全国のリソース不使用ランキング
 \item 使用料金
 \item 家族合計の表示
\end{itemize}


\section{評価}
友人10人にこのシステムを説明し，5段階でフィードバックを得た．
うち7人は良い印象を持ち，3人は普通と回答した．
（表\ref{eval}参照）

また，"これならエネルギー消費を普段から気にすることができる"
というコメントもあり，システムがユーザの意識を高める
効果があるということを確認した．

\section{まとめ}
本リポートでは，ユーザの環境問題への意識を高めるために
リアルタイムでリソースの使用上量をモニタリングする
システム案を提案した．また，評価ではシステムは効果的であろう
結果を得た．
